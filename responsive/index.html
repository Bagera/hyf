Step 2 - Mobile first

Here we have applied styles. It now works on mobile but as you can see when the screen is wider it becomes hard to read and the image gets very big

We use different unit types to achieve different goals

Units

em depends on the font-size. bottom: 1em; on the h1 will change if we change the font-size of the h1.

vw depends on the viewport width. font-size: 6vw; on the h1 will change if we change the width of the window.

% depends on the size of the parent. width: 100%; on the img will make it always be the full width of the header.

Other units include vh, rem and px. You can read more at css-tricks but don't take to much note of the cm, in and similar "real world" units.

Step 3 - Maxing out

The first thing we can do is to apply max-width: 600px; to our main and header. Adding margin: 0 auto; will center it for us and make it look better.

This will make the page a bit more legible but we still have some problems

Step 4 - Media queries & floats

The image was still a problem. It didn't look good. We can use media queries to make stuff change depending on screen width and more

We can use display: flex; and flex-direction: column-reverse; to do a little trick to change the order of elements.

We use float: left; to make the lists show side by side and we have to clear: both; makes the contact info appear as we would like.
